# 2026-05-27

## AI 协议重构：厂商绑定 → 协议分发

**背景**：原架构硬编码了 9 个 AI 厂商（OpenAI/Anthropic/Google/DeepSeek/智谱/月之暗面/豆包/MiniMax/custom），每个厂商映射独立的默认 base_url 和默认模型。

**改动**：简化为两种协议类型：`openai_compatible` 和 `anthropic_compatible`。

### 涉及文件

**Rust 后端（4 个文件）：**
- `src-tauri/src/ai.rs` — 删除 default_base_url() 和 default_model() 函数；call_ai() 改为按协议分发（if protocol == "anthropic_compatible"）；移除 json_object response_format 特殊处理；更新 test_connection/parse_image_jd 移除默认值回退
- `src-tauri/src/models.rs` — 更新 provider 字段文档说明
- `src-tauri/src/commands/ai_config.rs` — ai_config_create/ai_config_update 添加协议类型验证
- `src-tauri/src/db.rs` — 无需变更（provider 字段仍为 TEXT）

**前端（2 个文件）：**
- `src/app/api/ai-config.ts` — AIProvider 类型从 9 个厂商改为 `'openai_compatible' | 'anthropic_compatible'`
- `src/app/pages/AISettings.tsx` — 移除 providerMeta 厂商映射 + 9 卡片网格；替换为 protocolMeta（2 协议）+ 2 协议选择卡片 + 可选协议按钮切换；新增 endpointPresets 快速填充（8 个常用端点，按协议类型自动筛选）；base_url 和 model 改为必填字段

### 编译验证

- Rust: `cargo check` ✅ 成功
